# Release notes

One file per release, named `vX.Y.Z.md`. Written as part of the **Release
ritual** (see `CLAUDE.md`).

- The **canonical, durable record** is `project.md`'s Phase log. These notes are
  the human-readable, per-release companion — not a second source of truth.
- Scope: production releases tagged `vX.Y.0` (phase completions) and patch
  releases `vX.Y.Z`. The `v0.1.0-prototype` tag predates the product and has no
  note here.
- On each release: tag `vX.Y.Z`, update `project.md` + `README.md`, and add the
  matching `docs/releases/vX.Y.Z.md`.
