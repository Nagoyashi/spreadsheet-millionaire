# v0.13.0 — Backlog cleanup + Security hardening (2026-06-29)

A consolidation cycle: harden security for finance-app credibility and sweep the
highest-value backlog bugs. No new product surface — the app gets safer and more
resilient. Also folds in the post-v0.12 maintenance (dependency bumps + admin
portal review cleanups).

## Security hardening
- **Frontend security headers (#19)** — `vercel.json` now serves a strict
  **Content-Security-Policy** plus `X-Frame-Options: DENY`, `X-Content-Type-Options:
  nosniff`, `Referrer-Policy`, and HSTS on the document/asset routes (Talisman only
  covered `/api/*`). The app loads no external resources, so the CSP stays strict.
  This is the CSP backstop the CSRF-in-JS-memory design assumed but never had.
- **Bounded payloads + write rate limits (#20)** — `MAX_CONTENT_LENGTH` (256 KB)
  rejects oversized bodies with 413 before parsing; the saved-calculator `data`
  blob is capped (≤64 KB serialized, ≤24 deep, ≤1000 nodes → 422); calculator
  write routes carry per-user rate limits.
- **Session fixation (#34)** — the session id rotates on login/register, so a sid
  fixed in the browser pre-auth can't carry over (CSRF token preserved).
- **Login timing (#35)** — a dummy bcrypt comparison on the not-found path closes
  the email-enumeration timing channel.
- **Password cap (#36)** — passwords over bcrypt's 72-byte truncation point are
  rejected with a clear 422 (via the shared validator).

## Resilience / reliability
- **CSRF self-heal (#22)** — a stale-token 403 re-fetches a fresh CSRF token and
  retries the request once, so a long-lived tab heals instead of failing every
  mutation (the known "stale CSRF after restart" flake is now invisible).
- **Central 401 handling (#21)** — a 401 from any authenticated request resets the
  client auth state (user → null), so the UI reflects logged-out and prompts
  re-auth instead of generic save/load errors.
- **Render-error boundary (#23)** — a top-level `ErrorBoundary` wraps the route
  tree; a render throw (corrupt record, malformed permalink) shows a recoverable
  Reload / Back-to-app fallback instead of white-screening the SPA.

## Calculator bug fixes
- **Dividend growth (#77)** — the Dividend calculator's `dividend_growth` input was
  inert; the effective yield now compounds by it each year, driving the chart and
  summary.
- **Reachability display (#33)** — Emergency Fund and Debt Payoff showed the raw
  600-month safety cap as if it were the answer; they now show "Not reachable" /
  "Not payable", mirroring FIRE's "100+".
- **Sankey input validation (#24)** — a crafted `?data=` permalink (or a corrupt
  saved record) missing a group's `items` array crashed the render; a shared
  `normalizeSlices` now guards both load paths and falls back to a safe shape.

- **Dependency bumps** — backend (flask-cors, flask-limiter, bcrypt, marshmallow,
