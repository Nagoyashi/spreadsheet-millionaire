# v0.15.2 — Tracker debug + polish sweep (2026-07-15)

The full debug/UX list from the first day of real-world tracker use, shipped as
one patch (PR #308). One genuine data-loss bug, one "looks broken but is a
projection" confusion, and a set of workflow upgrades across all three tracker
surfaces — plus two new navigation placeholders.

## Overview tab
- **The Avg/Median toggle drives BOTH per-month cards.** Expenses previously
  showed the *year total* beside a *per-month* income figure — the mismatch
  that made Net and Savings rate look wrong at a glance. The math was verified
  correct (net = income − expenses; savings = net ÷ income); both are now
  labeled as full-year figures with formula subtext, and Expenses/month gets
  the same avg/median treatment over the same active-month set.
- **Income by category** — the pie card gains an Expenses/Income toggle.
- **Forecast months explain themselves.** Selecting a month whose chart bars
  are recurring-transaction *projections* (no recorded rows yet) now says so
  instead of showing a bare "no data" — the reported "filled out but empty"
  months were exactly these. Bar-click month index is coerced defensively.

## Monthly entry tab
- **Data-loss fix:** adding a category mid-entry wiped every unsaved amount —
  the mutation refetch flipped the page's loading state, which unmounted the
  grid. Mutations now refetch in the background; typed values survive.
- **Recurring transactions surface in the grid:** each affected category shows
  "+ $X recurring" (client-side projection for the selected month, anchor
  month excluded), and an explicit **"Apply recurring to empty fields"**
  button copies them into empty inputs only — never an accidental write
  (decision recorded; auto-prefill was rejected).
- **Category management grew up:** inline **rename** (the immutable key means
  history keeps resolving; case-insensitive name clashes 409), **drag-and-drop
  reorder** (`position` column; the endpoint takes the full id list per type
  and rejects stale lists wholesale), a **20-active-per-type cap** enforced
  server-side on create *and* restore with an in-grid warning, and the
  **Archived list collapsed** by default behind "Archived (N)".
- **Save month** is available at the top (month picker bar) and the bottom.
- **Comma decimals:** `NumInput` switched to `type="text"` +
  `inputMode="decimal"`, normalising `,` → `.` — "1,5" now works in every
  tracker field and calculator. Clamp semantics unchanged (DECISIONS.md
  § "Numeric input…" amended).

## Transactions tab
- **Layout:** filters → add/edit form → table — the form no longer hides
  below the full list.
- **Pagination:** 30 rows per page with numbered pages; any filter change
  snaps back to page 1.
- **Richer filters:** category, amount min/max (comma-tolerant), and date
  from/to, alongside the existing year/month/type — with a one-click
  "Clear filters".
