# v0.10.0 — Net Worth Tracker (2026-06-21)

The first tracker. SpreadsheetMillionaire grows beyond calculators: a full
**Net Worth** module where you record everything you own and owe across asset
categories, see your net worth and lifetime gain at a glance, and watch it trend
over time. It's **production-ready but ships dark** — hidden behind a feature
flag so production keeps showing "Coming soon" until a later waiting-list launch.
Built end-to-end (DB → API → UI), tested, and gated.

Modelled on the earlier `fintrackr_dev` Wealth module, adapted to this repo's
hard rules (Postgres `NUMERIC`/`%s`, `httpClient` over raw `fetch`, centralised
type enums, and `user_id` on every query).

## Data model (#100)
- Five normalised, user-scoped Postgres tables — `nw_assets`, `nw_liabilities`,
  `nw_investment_holdings`, `nw_real_estate`, `nw_snapshots` — with
  `NUMERIC(14,2)` money, `user_id` FK + index on each, and type enums enforced by
  named `CHECK` constraints.
- `backend/net_worth_types.py` is the single source of truth for the enum value
  sets (asset/liability/asset-class/property types), imported by both the schema
  and the DDL — the same pattern as `calc_types.py`.

## API (#101)
- `/api/net-worth/*` blueprint: per-resource CRUD for assets / liabilities /
  investments / real-estate, a SQL-aggregated `GET /summary` (category rollups,
  mortgage→liabilities, lifetime gain vs cost basis), and append-only
  `GET|POST /snapshots` with server-computed totals.
- A generic `NetWorthTable` data-access layer composes identifiers safely with
  `psycopg.sql`; **every query filters `user_id`** (Hard Rule #6). All routes
  `@login_required`; writes are CSRF-protected and rate-limited.

## Frontend (#102, #106, #107)
- `netWorthApi` (via `httpClient.createApi`, no raw fetch) + a `useNetWorthData`
  hook that fetches all resources + summary + snapshots and refetches on mutation.
- A tabbed Wealth page at `/app/net-worth` with a sticky Net Worth / Assets /
  Liabilities bar.
- **Category management** — a config-driven `CategoryManager` (one form + table
  for all five categories) with add/edit/delete, shared `NumInput`, and enum
  options mirroring the backend.
- **Overview dashboard** on recharts (already a dependency): allocation pie,
  category-breakdown bar, category cards with % of assets, lifetime gain + %
  return, and a net-worth-over-time line chart with a "take snapshot" action.

## Ships dark (#104)
- A single build-time flag, `NET_WORTH_ENABLED` (env `VITE_NETWORTH_ENABLED`,
  default `import.meta.env.DEV`), drives the published-tracker surface in
  `trackers.js` (`LIVE_TRACKERS` / `VISIBLE_UPCOMING`). When off (production), the
  route falls back to the "Coming soon" teaser and the nav link is hidden; on in
  dev/staging. The backend API stays available regardless — only the UI is gated.
- Net Worth ships free/auth-gated (no tier gate); the freemium decision is
  deferred to the later product-review cycle.

## Tests (#103)
- Backend: CRUD, summary rollups, snapshots, validation, and cross-tenant IDOR
  across all `nw_*` resources.
- Frontend: `@testing-library/react` added; component coverage for
  `CategoryManager`, `WealthPage`, and the recharts `Dashboard`, plus
  `netWorthApi` / helper unit tests.

## Bundled fix
- Session cookie is no longer marked `Secure` in development (Talisman default),
  which had silently broken local login in browsers that don't treat
  `http://localhost` as a secure context.
