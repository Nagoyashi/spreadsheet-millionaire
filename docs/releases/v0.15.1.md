# v0.15.1 — Custom categories + monthly-grid polish (2026-07-14)

A same-day patch on the v0.15.0 monthly grid, driven by first-use feedback:
the fixed category set doesn't survive contact with real bookkeeping. The
headline change makes categories the user's own; three smaller items polish
the surrounding surface. Shipped as one bundle (PR #303).

## Custom categories — add / archive / restore

Categories move from a fixed curated enum to a **user-scoped `ie_categories`
table** ("Strom + Gas", "Reisekosten" — whatever the user's bookkeeping
actually uses):

- **Add** inline in the Monthly-entry grid, per section; a new category is
  immediately usable in the grid, the transaction form, and breakdowns.
- **Archive is a recoverable soft delete.** An archived category leaves the
  pickers and the grid, but its history keeps aggregating in summaries, and
  its saved grid cells survive re-saves (shown read-only in an "Archived"
  list — the grid PUT's wholesale delete is scoped to *active* categories).
- **Duplicate-proof restore:** re-adding a name that case-insensitively
  matches an archived category **restores that row** instead of creating a
  duplicate, guaranteed by a unique index on `(user_id, type, lower(name))`.
- **No data migration:** defaults are lazily seeded per user with keys equal
  to the old curated slugs, so every historical `ie_transactions.category`
  value resolves. Keys are immutable server-generated slugs (collisions get
  `-2` suffixes), leaving rename cleanly addable later.
- Category **validity moved from schema `OneOf` / DB CHECK to model-layer
  checks** against the user's active set (a static schema can't see per-user
  data); the old union CHECK became a plain length bound.
- `ie_categories` joined `USER_SCOPED_TABLES` (deletion cascade + data
  export) — the #179 drift guard caught the initial conftest gap in CI,
  exactly as designed. Decision + supersede note in `DECISIONS.md`
  § "Income & Expense Tracker" (Custom categories).

## Color-coded grid fields

`NumInput` gained an optional `tone` prop (tinted focus ring, `$` prefix, and
filled-field background). The grid's income section renders emerald, expenses
rose — filled cells read at a glance. Calculators keep the neutral default.

## "Bulk upload" tab (coming soon)

Fourth tab on the I&E page: a static teaser for bank-statement import — CSV
(roadmapped `v0.16.1`, pre-filed #206–#210) and PDF/Kontoauszug extraction
(backlog #296) — pointing users at Monthly entry meanwhile. No upload surface
is wired.

## Overview year selector fix

The Cashflow year dropdown only listed years that already had data, so with a
single data year there was nothing to switch to ("can't select the year"). It
now offers a window around the current year as well, matching the
Monthly-entry picker — empty years render zeroed, ready for back-filling.
