# v0.15.3 — Marketing redesign: light fintech landing + app sidebar (2026-07-15)

The public marketing page and the in-app sidebar move from the dark stone-950 + amber theme to a light, neutral fintech design (white / gray-50 / blue-600, amber retained as the brand accent), per the founder's high-fidelity design handoff. Copy rewritten to be calm and trustworthy for personal-finance beginners. Plus: the publish defaults now mirror the completed build-in-public rollout.

## Marketing landing page (`/`) — full redesign

- **Nav** — light sticky bar (`bg-white/88` blur, gray-200 rule): gray section links, blue-600 "Open app" button. The anonymous "Login App" hover menu stays, restyled to a white panel with shadow.
- **Hero** — new copy ("Understand your money. Plan what comes next."), beta/open-source badge pill, blue primary + outline secondary CTAs, amber-check trust row.
- **`HeroAppPreview`** (new) — an interactive live-DOM miniature of the app (1060×420): sidebar with three clickable views (Compound Interest, Net Worth, Income & Expenses), each with inline-SVG charts; scales down responsively via ResizeObserver.
- **Calculator showcase** — light cards (tinted icon tiles in each calculator's color family, gradient bar, "Open →" row) on the existing registry-driven `Carousel`; white circular arrows; two "TRACKER · LIVE" cards linking to the trackers.
- **Value props** ("Built to be trusted") — gray-50 band, amber icon tiles, four honest-tooling props.
- **Four surfaces** ("More than calculators") — Guide / Calculators (LIVE) / Comparison / ETFs & Stocks, dashed borders on coming-soon cards.
- **Dark CTA band** (new) — slate-900 with a decorative emerald growth curve + amber dashed target line; "Your numbers are waiting."
- **Footer** — light restyle, structure/links unchanged.
- `LegalLayout` + coming-soon pages share the nav/footer, so their canvases flipped to light too (structure and copy untouched).

## In-app (`/app`)

- **`AppSidebar`** — the old dark gray-900 gradient panel is now white with a gray-200 rule: blue-50/blue-700 active items, amber only in the wordmark, blue-600 "Create account" in the anonymous footer. Collapsed icon rail matches. `UserFooter` and `SavedCalculationsSidebar` (which render inside it) restyled with it; the mobile drawer gets a shadow for separation. All behavior unchanged.
- Sidebar wordmark sized to fit (was truncating next to the collapse chevron).
- The "Free while in beta" badge is gone from the calculators landing hero.

## Publish defaults mirror production (rollout complete)

All 12 calculators and both trackers have been live in production (admin-toggled); the build-time defaults now say so too:

- `registry.js`: all entries `published: true`; the flag is now only the offline fallback / fresh-DB seed.
- `usePublished.js`: the offline fallback includes the tracker slugs (via a `published` field on `upcomingFeatures.js` entries), so a failed `/api/calculators/published` fetch still renders the full production surface.
- Backend: `DEFAULT_PUBLISHED_TYPES` = all calculators; `DEFAULT_PUBLISHED_PUBLISHABLE` (new seed source for `db_init`) adds both trackers. Existing DBs are untouched — seeding stays `ON CONFLICT DO NOTHING`; the admin toggle remains the runtime source of truth.
- `test_admin.py` updated (tracker toggle test now proves unpublish → republish); DECISIONS.md §§ "MVP narrowing" / "Runtime publish state" note the completed rollout.

Suite: 237 frontend tests across 35 files; backend admin/publish tests adapted.
