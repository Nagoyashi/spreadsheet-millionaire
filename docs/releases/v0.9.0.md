# v0.9.0 — Test & CI foundation (2026-06-20)

The automated safety net: backend (pytest) and frontend (vitest) test harnesses,
coverage of the highest-risk surfaces (auth flows, tenant isolation, saved-data
migration), linting/formatting, and a CI gate that actually runs them on every
PR. No user-facing product changes — this cycle hardens the foundation a live
product needs before the next features land.

## Test harnesses
- **pytest harness** (#25): `create_app()`-factory fixtures with a *forced*
  hermetic env (a test run can never reach the prod database, Redis, or email);
  `app`/`client`/`get_csrf_token` fixtures and a truncation-isolation `db`
  fixture gated on `TEST_DATABASE_URL`.
- **vitest harness** (#26): jsdom environment wired into `vite.config.js`,
  `npm test` script.

## High-risk coverage
- **Saved-data migration engine** (#30): 21 vitest cases — Sankey v1→v2 reshape,
  idempotency, legacy-as-v1, forward-compat skip, `strip`/`injectVersion`.
- **Auth flows end-to-end** (#29): 25 pytest cases — register/login/logout,
  forgot/reset (no enumeration, token-first, single-use, sibling invalidation),
  delete/change-password/change-email; email mocked.
- **IDOR / tenant isolation** (#28): 10 pytest cases guarding Hard Rule #6 —
  user B can't read/update/delete user A's saved calculators, at the route and
  model layers; unauthenticated access is 401.

## Lint, format, CI
- **ESLint (flat) + Prettier** (#41): React + hooks rules, a custom guard banning
  raw `fetch` outside `src/api/` (Hard Rule #4). Surfaced and fixed a real
  `rules-of-hooks` bug in `CalculatorPage.jsx` (an early return before 10 hooks
  could crash React) and removed dead code (`DividendCalculator`'s unused
  `dividend_growth` → tracked in #77).
- **CI gate** (#27): runs frontend lint/test/build + backend pytest on every PR
  to `develop`/`main`. Caught and fixed two false-greens — the backend suite was
  being silently skipped, and `cd backend && pytest` was broken — and added a
  Postgres service so the DB-backed tests actually run (no Neon reliance).

## Security
- **Leaked-credential cleanup** (#16): removed the on-disk credential dump,
  confirmed it was never in git history, and rotated the Neon DB password and
  Upstash Redis token.

## Release plumbing
- `release.yml` gains `issues: write` so the tag-triggered workflow can close the
  cycle milestone (milestones live under the Issues API) — without it the
  milestone-close step would 403.
