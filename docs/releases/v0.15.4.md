# v0.15.4 — Guide, Comparison & ETFs drafts behind superadmin preview (2026-07-16)

The three remaining marketing surfaces — Guide, Comparison, ETFs & Stocks — now exist as full draft pages in the bundle, visible **only to superadmins**. Public visitors keep the coming-soon pages, unchanged. This lets the surfaces be built and reviewed in production without shipping them.

## The three draft pages (placeholder content, per the design handoff)

- **Guide (`/guide`)** — SEO education hub: hero, search + category chips (client-side filtering over placeholder articles), featured-article card, 3-col article grid with color-coded category pills, "Watch instead" video strip, newsletter band (visual no-op pending a provider decision).
- **Comparison (`/comparison`)** — affiliate comparisons with the trust layer: the affiliate-disclosure pill above the fold (→ `#how-we-rate`), four category tiles, the brokers comparison table with the warm-tinted "OUR PICK" row and amber rating bar, `rel="sponsored nofollow"` on affiliate placeholders, "How we rate" section.
- **ETFs & Stocks (`/etfs-stocks`)** — instrument lookup shell: search, filter pills (static), category quick-links, results table with placeholder funds (type pills, signed emerald/rose 1Y/5Y), load-more row, calculator cross-sell band → `/app`. No market-data API yet — all data is clearly-marked local placeholder.

## Superadmin gating

- New `SuperadminPreview` route wrapper in `App.jsx`: `auth.user.is_superadmin` → the draft page (with a slim amber "Superadmin preview" banner under the nav); everyone else — including plain admins — the untouched coming-soon page. UI-layer gating is deliberate and documented in-code: static content preview, no backend data (vs. hard rule #8's three-layer rule for paid features).
- The landing page's "More than calculators" statuses stay **COMING SOON**; each flips to LIVE only when its surface ships publicly (ship checklist: unwrap the route, remove the banner, flip the card).
