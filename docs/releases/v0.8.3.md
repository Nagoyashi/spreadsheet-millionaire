# v0.8.3 — Release automation + process docs (2026-06-17)

Docs and CI only; no application changes (no `backend/` or `frontend/` code).

- **Tag-triggered release workflow** (`.github/workflows/release.yml`): pushing
  an annotated `vX.Y.Z` tag now publishes the GitHub Release from
  `docs/releases/<tag>.md` and closes a matching milestone — automating the
  previously manual, easily-forgotten step. Fails loudly if the notes file is
  absent on the tagged commit.
- **Session protocol + release-cycle state machine** added to `CLAUDE.md`: on any
  cold start, live cycle state is derived from GitHub (the open milestone),
  never guessed or remembered.
- **Milestone-based cycles:** one open milestone = the current cycle; `project.md`
  carries a mirrored *Current cycle* pointer (the milestone wins on disagreement).
- **Doc-ownership map** consolidated in `README.md` § "Where to learn more";
  `STATUS.md` / `project.md` / `CLAUDE.md` now link to it instead of restating it.
- First `develop`→`main` merge to use the ritual — arms `release.yml` on `main`
  for every future production release.

→ `CLAUDE.md` §§ "Session protocol", "Release ritual" · `project.md` Current cycle.
